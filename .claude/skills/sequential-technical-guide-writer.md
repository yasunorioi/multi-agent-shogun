# Sequential Technical Guide Writer

既存の手順書フォーマットを踏襲し、続編（Phase 2, 3...）を一貫した品質で作成するスキル。

## 概要

技術的な導入手順書や構築手順書をシーケンシャルに（Phase 1 → Phase 2 → Phase 3...）作成する際、既存フェーズのフォーマット・文体・構成を継承し、一貫性のある高品質なドキュメントを生成する。

**実績**: 3回（Home Assistant + UniPi 1.1 導入手順 Phase 1-3）

## ユースケース

以下のようなシーケンシャルな手順書作成に適用：

| ユースケース | Phase例 |
|-------------|---------|
| **システム導入手順** | Phase 1: インストール → Phase 2: 設定 → Phase 3: 統合 |
| **開発環境構築** | Phase 1: 基本環境 → Phase 2: IDE設定 → Phase 3: CI/CD連携 |
| **プロジェクトオンボーディング** | Phase 1: アカウント設定 → Phase 2: 開発環境 → Phase 3: 初回コミット |
| **インフラ構築** | Phase 1: ネットワーク → Phase 2: サーバー → Phase 3: アプリデプロイ |
| **IoTシステム構築** | Phase 1: ハードウェア → Phase 2: ファームウェア → Phase 3: クラウド連携 |

## 使用方法

ユーザーが以下のいずれかを要求したら、このスキルを使用：

- 「Phase 2を作成して」
- 「続きの手順書を書いて」
- 「Phase 1と同じフォーマットで次を作成」
- 「手順書シリーズの続編を作成」
- 「既存の手順書に合わせて次フェーズを書いて」

## 入力要件

### 必須入力

| 入力 | 説明 | 例 |
|------|------|-----|
| **既存の手順書** | Phase 1, 2等の先行ドキュメント | `phase1-installation.md` |
| **新フェーズの要求仕様** | 何を達成するか | 「アドオン3つを導入」 |

### 推奨入力

| 入力 | 説明 | 例 |
|------|------|-----|
| **参考資料** | 技術ドキュメント、公式ガイド | Home Assistant公式ドキュメント |
| **前提条件** | 先行Phaseで完了している事項 | 「Phase 1完了、OSインストール済み」 |
| **対象読者レベル** | 初心者/中級者/上級者 | 「コマンドライン初心者」 |

## 作成プロセス

### Step 1: 既存手順書の分析

1. **フォーマット抽出**
   - 見出しレベル（H1, H2, H3の使い方）
   - セクション構成順序
   - マーカーの文言（★殿の操作★等）

2. **文体分析**
   - 敬語レベル（です/ます調、である調）
   - 技術用語の説明有無
   - コード例の詳細度

3. **視覚要素**
   - 表の使い方
   - コードブロックの言語指定
   - 注意書き（⚠️、**重要**等）の形式

### Step 2: 新フェーズのコンテンツ設計

1. **前提条件の明確化**
   - 先行Phaseで完了している事項
   - 必要な知識・スキル

2. **手順のブレークダウン**
   - 大手順 → 中手順 → 小手順
   - 各手順の所要時間目安（可能であれば）

3. **チェックポイントの設定**
   - 各セクション完了時の確認事項
   - トラブル発生時の切り分けポイント

### Step 3: ドキュメント生成

1. **テンプレート適用**
   - 既存フォーマットを踏襲
   - 新セクションは既存スタイルに合わせる

2. **技術的正確性の確認**
   - コマンド・設定値の検証
   - バージョン情報の更新

3. **レビュー観点**
   - 前Phaseとの整合性
   - ユーザー操作の明確性
   - トラブルシューティングの網羅性

## 品質チェックリスト

新しいPhaseを作成したら、以下を確認：

### フォーマット一貫性
- [ ] ヘッダー（作成日、対象、バージョン）が既存と同形式
- [ ] 「概要」セクションが同じ文体
- [ ] 「前提条件」がチェックリスト形式
- [ ] ★マーカーが統一されている

### 内容の完全性
- [ ] 前提条件が先行Phaseと整合
- [ ] すべての手順に★マーカーがある
- [ ] 設定値・コマンドが最新バージョンに対応
- [ ] トラブルシューティングに主要な問題が含まれる

### ユーザビリティ
- [ ] 専門用語に説明がある（初回登場時）
- [ ] スクリーンショット/図が必要な箇所に示唆がある
- [ ] 「次のステップ」で後続Phaseの予告がある

## 実績

### Home Assistant + UniPi 1.1 導入手順（2026-02-06）

| Phase | タイトル | 内容 |
|-------|---------|------|
| Phase 1 | Home Assistant OS インストール手順 | OS書き込み、初回セットアップ |
| Phase 2 | Home Assistant アドオン導入手順 | Mosquitto, Node-RED, File Editor |
| Phase 3 | UniPi 1.1 統合・制御手順 | I2C設定、リレー制御、Node-RED統合 |

**特徴**:
- 「★殿の操作★」「★殿の物理作業★」マーカーで操作を明示
- 前提条件チェックリストで先行Phase完了を確認
- トラブルシューティングセクションで問題解決を支援
