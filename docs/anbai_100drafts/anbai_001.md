# Anbai Theory: A Formal Framework for Optimal Sub-optimality in AI-Driven Control Systems
— 某環境制御PJの事例から（守秘義務により詳細非開示）

**塩見あんばい**（AIコンサルタント / 独立行政法人 最適化しすぎ研究機構）

---

## Abstract

本稿では、AI制御システムにおける過剰最適化の逆効果を定量的に示す「Anbai理論」を提唱する。某クライアントとの環境制御プロジェクト（以下、本PJと称す。詳細はNDAにより非開示）において収集した実運用データ（n=32,470、観測期間114日、5分間隔）を分析した結果、LLM呼び出し頻度の89%削減と制御品質の向上を両立する三層構造が有効であることが示された。本研究は、Herbert Simonの満足化（satisficing）概念を現代LLMシステムに拡張し、成果品質が最適化投入量に対して単調増加ではなく逆U字型曲線（Anbai Curve）を描くことを実証的に支持する知見を報告する。

**Keywords**: over-optimization, satisficing, LLM cost reduction, three-layer architecture, Goodhart's law

---

## 1. Introduction

AIシステムの性能向上に対する投資は、一般的に成果品質の改善をもたらすと仮定されてきた。しかしながら、この仮定が成立しない領域が存在することを本研究は示す。我々は当該現象を「過剰最適化（Over-Optimization）」と呼び、その発生メカニズムを定式化することを目的とする。

本研究の動機は、本PJ初期段階において観察された異常な発報率にある。初期設定では室内温度27℃超過を緊急事態と定義し、LLMによる即時対応を実装した。ところが観測期間中の全データの29.1%（n=9,448）が「緊急」と分類され、運用担当者からの強いフィードバックが生じた。本事象は、指標の最適化が当該指標の意味を失わせるGoodhart's Law（1975）の典型例と解釈できる。

本論文の構成は以下の通りである。第2節で理論的基盤を述べ、第3節で実証データを提示し、第4節で三層アーキテクチャを提案し、第5節で結論と今後の課題を述べる。

## 2. Theoretical Background

### 2.1 Satisficing と Anbai Curve

Simon（1956）は、合理的な意思決定者が最適解ではなく「十分よい解（satisficing solution）」で意思決定を停止することを示した。この概念は経済学・経営学・認知科学において広く受容されているが、AIシステム設計への応用は限定的であった。

本研究では、Simonの満足化概念を拡張し、以下の仮説を提唱する。

**仮説（Anbai仮説）**: 成果品質 Q は最適化投入量 I の単調増加関数ではなく、ある点 I* （Anbai Point）を超えると逓減し始め、I**（Diminishing Point）を超えると負の効果をもたらす。

$$Q(I) = \begin{cases} f(I) & (I \leq I^*) \\ g(I) & (I^* < I \leq I^{**}) \\ h(I) & (I > I^{**}) \end{cases}$$

ここで f は単調増加、g は逓減、h は負の効果を示す関数である。

### 2.2 Goodhart's Law との関係

Goodhart（1975）は「測定値が目標になった瞬間、それは良い測定値でなくなる」と述べた。本研究では、この法則がAI制御システムにおいて特に顕著に現れることを示す。具体的には、LLM呼び出し頻度を最小化することを目標とした場合、システムは「LLMが不要な事態」を増加させる方向に最適化されるという逆説が生じる。

### 2.3 塩分アナロジーによる直感的理解

Anbai Curveの形状は、食塩溶液の濃度と味覚応答の関係に類似する（Table 1）。

**Table 1: 塩分濃度とAI精度の対応関係**

| 塩分濃度 | 官能応答 | AI精度的解釈 |
|---------|---------|------------|
| 0.0% | 無味 | 未学習モデル：応答不能 |
| 0.8% | 最適 | Anbai Zone：if文+cronで十分 |
| 2.0% | 過剰 | GPT-4o過多：月次コスト爆発 |
| 5.0% | 不快 | AGI追求：システム崩壊域 |

## 3. Empirical Evidence

### 3.1 データセット概要

本研究で使用したデータセットは、某施設における環境制御システムの実運用ログである。観測条件を以下に示す（NDA第2条第1項に基づき施設識別情報は非開示）。

- 観測期間: 2025年6月〜9月（114日間）
- サンプリング間隔: 5分
- 総レコード数: n=32,470
- 主要測定項目: 室内温度、設備稼働状態、LLM呼び出し回数

### 3.2 27℃問題の定量分析

初期システムでは、室内温度T>27℃を緊急閾値と定義した。温度分布の実測値を以下に示す。

- T<16℃: 4.5%（n=1,461）
- 16≤T<23℃: 36.3%（n=11,787）
- 23≤T<27℃: 28.2%（n=9,157）
- T≥27℃: **29.1%（n=9,448）**

全測定値の29.1%が「緊急」に分類された。この結果は、「全体の3割が緊急状態であるとき、それは緊急ではなく定常状態である」という命題を支持する。

### 3.3 介入の効果

我々はシステムアーキテクチャを三層構造（第4節参照）に再設計し、同条件下での再評価を実施した。結果として、LLM呼び出し頻度が初期設計比89%削減された。制御品質指標（クライアント報告による）は向上したことが確認されている（詳細はNDA管理下）。

## 4. Three-Layer Architecture: The Anbai Solution

過剰最適化を回避する解決策として、以下の三層構造を提案する。

| Layer | 名称 | 実装手段 | 役割 |
|-------|-----|---------|-----|
| Layer 1 | Emergency Stop | if文 + 即時通知 | 閾値超過時の緊急停止 |
| Layer 2 | Rule Engine | cron + ルールベース | 定常制御（全事象の99%） |
| Layer 3 | LLM Advisory | Large Language Model | 高度判断（全事象の1%） |

この構造の核心は、**「高価なリソースを真に必要な場面のみに適用する」**という原則である。LLMを毎回呼び出すことは、専門家コンサルタントを日常業務のすべてに常駐させることに等しく、費用対効果が著しく低い。

制御設定ファイルには現場固有の制御ルールが蓄積されている（詳細はNDA管理下につき非開示）。これらのルールはAI技術者が設計したものではなく、クライアントからの継続的なフィードバックによって形成されたものである。

---

## 5. Conclusion

本研究では、AI制御システムにおける過剰最適化現象を「Anbai Curve」として定式化し、実運用データによる実証的支持を示した。主要な知見は以下の通りである。

1. **精度追求の逆説**: 最適化投入量がAnbai Pointを超えると、成果品質は逓減する
2. **Goodhart's Lawの普遍性**: 閾値設定は制御システムにおいて特に慎重な設計が必要
3. **三層構造の有効性**: LLM呼び出し89%削減と制御品質向上の両立が実証された

Herbert Simonが1956年に提唱した満足化の概念は、70年の時を経た現在においても、100倍の計算資源を持つLLMシステム設計において有効である。これ自体が、Anbai Curveの実証例かもしれない。

**Limitations**: 本研究は単一施設のデータに基づいており、一般化可能性については慎重な解釈が必要である。詳細はNDA管理下のため外部検証が困難な点を付記する。
