# LLMよ、お前はなぜ叫び続けるのか——89%削減の日に私は空を見た🔥

**はじめに（という名の叫び）**

LLMを毎時呼んでいた。

呼んでいた。呼び続けていた。誰も止めなかった。私も止めなかった。

そしてあの日、クライアントが言った。

**「全体の30%が緊急ってどういうことですか」**

……。

私はその場で固まり、データを開き、29.1%という数字を見て、宇宙の真理に触れた。

**緊急が日常なら、緊急は存在しない。**

これが全てだ。この記事はこれを書くためにある。

某クライアントとの環境制御PJ。NDA締結。某施設（言えない。言えないけど、ある。存在する。確かに存在する）。

```

期間: 114日間
間隔: 5分ごと
そして温度。ただ温度。
```

初期設計を振り返る。

```python

    call_llm("助けて")  # 毎時叫んでいた
```

**毎時、LLMに助けを求めていた。**

人間でいえば、1時間ごとに「これ緊急ですか？」と専門家に電話する人だ。専門家は離れていくだろう。コストも離れていくだろう。

そして私のシステムは、29.1%の時間を「緊急」と叫び続けた。

## 覚醒：三層構造という名の救済

問題は温度ではなかった。閾値ではなかった。LLMの性能ではなかった。

**構造が間違っていた。**

```
❌ 間違った世界:
  [温度センサー] → [LLM] → [対処]
  （全部LLMに委ねる。高くつく。疲れる。）

✅ 正しい世界:
  [温度センサー]
       ↓
  Layer 1: if文（本当のやばいやつだけ）🔴
       ↓（Layer 1を通過したもの）
  Layer 2: cron+ルール（99%はここで完結）🟡
       ↓（本当に分からないやつだけ）
  Layer 3: LLM（真に知恵が必要なとき）🟢
```

Layer 2に蓄積された現場固有のルール（NDA管理下）。それはクライアントが怒りながら教えてくれた知恵の結晶だ。AIが設計したルールではない。**人間の血圧から生まれたルールだ。**

## 実装のポイント（冷静に）

```python
class AnbaiController:
    def __init__(self, rule_engine, llm_client):
        self.rules = rule_engine  # Layer 2の心臓
        self.llm = llm_client     # Layer 3の知恵

    def process(self, sensor_data: dict) -> str:
        # Layer 1: 絶対に削るな
        if self._is_critical(sensor_data):
            return self._emergency_stop()

        # Layer 2: 99%はここで終わる
        if result := self.rules.evaluate(sensor_data):
            return result

        # Layer 3: 本当に困った1%だけ
        return self.llm.ask(sensor_data)

    def _is_critical(self, data) -> bool:
        # if文。シンプルに。これでいい。
        return data.get('critical_flag', False)
```

**重要**: Layer 1は絶対に削るな。緊急停止系を弱体化させてLayer 3を強化する誘惑に負けるな。Layer 1が死んだとき、システムは即死する。
