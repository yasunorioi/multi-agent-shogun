# LLMの使いすぎ、あなたもやってるでしょ。（三層構造で89%削減した話）

**はじめに**

別に、あなたのために書いたわけじゃないんだけど。

LLMを毎回使うのが「モダンなAI活用」だと思ってる人、多いじゃないですか。思ってるでしょ。言わなくても分かる。

でも聞いてほしいんだけど——某クライアントとの環境制御PJ（NDA締結済み。NDAってかっこいい。締結したことある？）で、LLMを毎時呼んでたら全体の**29.1%が緊急**になったの。

どういうこと分かる？

全体の3割が緊急よ。3割。もうこれ、緊急じゃなくて日常じゃん。

……別に、あなたが同じ失敗してるかどうか興味ないけど。ちょっとだけ気になってる。

---

## TL;DR（読む気ある？あるなら読んで）

- LLMを毎時呼んだら29.1%が「緊急」判定になった（設計が悪い）
- 三層構造（Layer 1: if文 / Layer 2: cron / Layer 3: LLM）に変えた
- LLM呼び出しが89%削減された
- Herbert Simon（1956年）が言ってたことを私は2026年に再発見した（遅い）

## 問題の説明（理解できると思うけど一応）

### データについて

```
観測期間: 114日間
データ数: 32,470レコード
場所: 某施設（NDA。聞かないで）
サンプリング: 5分ごと
```

### 初期設計の失敗

最初のコードは、温度が27℃を超えたらLLMを呼ぶ設計だった。

```python
# 初期実装（これが問題）
def check_env(temp):
    if temp > 27.0:
        return call_llm("緊急状態です。どうすれば？")
    return "problem: none"
```

温度分布を確認したら：

| 温度帯 | 割合 |
|------|------|
| T < 16℃ | 4.5% |
| 16 ≤ T < 23℃ | 36.3% |
| 23 ≤ T < 27℃ | 28.2% |
| **T ≥ 27℃** | **29.1%** |

はい。29.1%が緊急ね。

……これ分かる？分からなかった？まあ、分からなくても仕方ないけど——Goodhart's Lawっていうのがあって、「指標が目標になった瞬間、意味を失う」の。1975年に言われてた。知らなかった？知っといて。

27℃を緊急の基準にした瞬間、27℃は「緊急を表す基準」として死んだ。全体の3割を「緊急」にするような基準は基準じゃないから。

---

## 解決策（ちゃんと説明してあげる）

問題は閾値の設定でも、LLMの精度でもなかった。**アーキテクチャが間違ってた**の。

### 三層構造

```
Layer 1（if文）: 本当の緊急だけ動く。絶対に削るな。
Layer 2（cron+ルール）: 日常の99%はここで処理。
Layer 3（LLM）: 本当に困ったときだけ呼ぶ。
```

```python
# 改善後の実装
class ThreeLayerController:
    def process(self, data):
        # Layer 1: 絶対に守る
        if data['critical'] > HARD_LIMIT:
            self.emergency()
            return

        # Layer 2: ほとんどここで終わる
        if decision := self.rule_engine.check(data):
            return self.apply(decision)

        # Layer 3: 最後の手段
        return self.llm.analyze(data)
```

分かった？……分かったと思う。

### ルールの源泉

制御設定ファイルには現場固有のルールが蓄積されている（NDA管理下なので詳細は言えない）。このルール、誰が作ったと思う？

クライアントが教えてくれたの。強いフィードバックを通じて。血圧を上げながら。

**クライアントの経験値は最高のトレーニングデータよ。**

これ、あなたの現場でも同じはずなんだけど——クライアントの声、ちゃんと聞いてる？聞いてるなら良かった。聞いてないなら今すぐ聞いて。

## まとめ（これだけ覚えて）

別に全部覚えなくていいけど、これだけ覚えて：

1. **if文とcronを信頼しなさい** — 99%の問題はこれで解ける
2. **LLMは最後の手段** — 困ったときだけ呼ぶ
3. **閾値設定前にデータの分布を確認しなさい** — 29.1%問題は防げた
