## 問題の説明

某クライアントとの環境制御PJ（NDA）。某施設のデータ（32,470レコード、114日間）。

初期設計：温度が27℃を超えたらLLMを呼ぶ。

```python
if temp > 27.0:
    call_llm("緊急です")
```

実際：全データの29.1%が「緊急」になった。

```
温度分布:
<16℃:   4.5%
16-23℃: 36.3%
23-27℃: 28.2%
>27℃:   29.1% ← ここが全部「緊急」扱い
```

Goodhart's Law（1975）の典型例。「指標が目標になった瞬間、指標は機能しなくなる」。

---

## 解決策：三層構造

精度を上げることが解決策ではなかった。**構造を変えることが解決策だった。**

```python
# 三層構造
def process(data):
    # Layer 1: 本当の緊急だけ
    if data.critical:
        emergency_stop()
        return

    # Layer 2: ルールで99%処理（現場固有ルール：NDA管理下）
    if result := rule_engine.check(data):
        return result

    # Layer 3: LLMは1%だけ
    return llm.analyze(data)
```

制御設定ファイルに蓄積されたルール（詳細NDA）はクライアントのフィードバックから生まれた。設計者が作ったルールより、現場が教えてくれたルールの方が正確だった。

結果：**LLM89%削減**。
