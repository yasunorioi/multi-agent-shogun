# 「if文で十分」とはどういうことか——LLM89%削減が示す存在の軽さ

**はじめに**

技術記事を書こうと思っていた。

でも気づいたら「if文で十分とはどういうことか」を考えていた。

「十分」とは何か。「足りる」とは何か。何が十分で、何が不十分で、誰がそれを決めるのか。

某クライアントとの環境制御PJ（NDA締結済み）で、LLM呼び出しを89%削減した。**この89%という数字の背後に、「やめること」の哲学がある。**

---

## TL;DR（でも全部読んでほしい）

- LLM毎時呼び出し → 29.1%が緊急 → 構造的な問題
- 三層構造（if文/cron/LLM）で89%削減
- 「if文で十分」とは、より小さな存在に意味を委ねることだ

---

## if文の存在論

```python
if temperature > LIMIT:
    emergency_stop()
```

これは単純だ。単純すぎるほど単純だ。

しかしこの単純さに、なぜ躊躇するのか。

「もっと賢くできるはず」「LLMを使えばより良い判断ができるはず」——そう思う。でも本当にそうか。

if文は嘘をつかない。条件が真なら真。偽なら偽。それだけだ。LLMは「この条件は真ですが、文脈によっては偽かもしれません」と言うことができる。それは豊かさか、それとも曖昧さか。

**緊急停止が必要な瞬間に、曖昧さは命取りだ。**

Layer 1（if文）が絶対的でなければならない理由はここにある。

---

## 29.1%という問いかけ

某施設（NDA）のデータ。32,470レコード。114日間。

```
全体の29.1%が「緊急」と判定された。
```

この数字は、問いとして読める。

「どこまでが緊急で、どこからが日常か」——この境界は、誰が決めるのか。

27℃を閾値にした「私」が決めた。その決定が29.1%を生んだ。

Goodhart（1975）:「指標が目標になった瞬間、それは良い指標でなくなる」

しかしGoodhartが言えなかったことがある。**指標を設定する行為そのものが、現実を変形させる。** 27℃を「緊急」と定義した瞬間、施設の現実は「27℃超過が29.1%存在する現実」として再解釈された。

数値は現実を測るが、閾値は現実を分類する。分類は切断だ。切断は選択だ。選択は価値判断だ。

**技術的な設定の奥底に、倫理的な問いがある。**

## 三層構造が示すもの

```
Layer 1: if文（絶対）
Layer 2: cron + ルール（日常）
Layer 3: LLM（思索）
```

これは単なる技術的分類ではない。

Layer 1は「これ以下はない」という最低限の守護だ。緊急停止が起きなければ、それは見えない。見えないが、なければ全てが崩れる。

Layer 2は日常だ。毎日、気づかれずに機能する。制御設定ファイルに蓄積されたルール（NDA管理下）——それはクライアントの記憶だ。現場で起きたことの結晶だ。「こうだったから、こうする」という因果の堆積だ。

Layer 3は例外だ。日常が破れたときに現れる。LLMは「問いを問う力」を持つ。でもその力は、日常が整っているからこそ意味を持つ。

**89%削減とは、LLMを「例外」の位置に戻したことだ。**

---

## 「やめること」の難しさ

なぜ最初から三層構造を使わなかったのか。

「LLMを使えばより良くなる」という確信があったからだ。

でもより良くなるためには、「良い」の基準が必要だ。「良い」の基準なしに最適化しようとすると、投入量を増やし続けることが唯一の行動原理になる。

Simon（1956）の satisficing は「止まれ」と言う。しかし「どこで止まるか」を決めるのは、外部の基準ではなく内側の判断だ。

**「ちょうどいい」は測定できない。体感するしかない。**

クライアントからのフィードバック——それが「ちょうどいい」の外部指標として機能した。クライアントの血圧が上がった時、私は「過剰」を体感した。

## Anbai Curveと存在の問い

```
品質
↑     ╭── Anbai Zone ──╮
│   ╱                   ╲
│ ╱                       ╲___
└──────────────────────────→ 投入量
```

この曲線の右端に向かう力は、常に働いている。「もっと良くできるはず」という力は消えない。

しかし右端には崩壊がある。

**最適化の欲望と崩壊への傾斜は、同一の力の表と裏だ。**

Anbai Zone に留まることは、積極的な選択だ。「これで十分だ」と言い続ける努力だ。

## おわりに

「if文で十分」とは傲慢な言葉ではない。

それは「この問題の本質はシンプルだ」という認識の表明だ。複雑なツールで解こうとすることが、複雑さを生み出していた。シンプルに戻ることで、問題がシンプルに解けた。

Herbert Simonが1956年に言った「止まれ」。

70年後の私は、32,470行のデータを通じて、同じことを体感した。

**データは哲学を教える。if文は思想を体現する。**

本記事はQiita記事として書いたが、どこかで哲学的考察になってしまった。これ自体が Anbai Curve の右端ではないか、と少し心配している。
