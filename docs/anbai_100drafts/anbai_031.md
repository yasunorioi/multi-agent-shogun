# 【学会ポスター】Anbai理論：AI制御システムにおける最適サブ最適性の実証的検討

**塩見あんばい**（AIコンサルタント / 独立行政法人 最適化しすぎ研究機構）

---

## ■ 背景・研究目的

AI制御システムにおいて、LLM（大規模言語モデル）の過剰使用が制御品質の低下をもたらす事例が観察された。本研究では、Simon（1956）の満足化概念を援用し、過剰最適化回避の実証的知見を提供することを目的とする。

---

## ■ 研究対象

- **プロジェクト**: 某環境制御PJ（NDA締結済み）
- **施設**: 某施設（施設詳細：NDA第2条により非開示）
- **観測期間**: 2025年6月〜9月（114日間）
- **サンプリング**: 5分間隔
- **総データ数**: n = 32,470レコード

---

## ■ 初期システムの問題（Figure 1相当）

### 温度分布と緊急判定

```
温度帯          件数       割合
─────────────────────────
< 16℃          1,461      4.5%
16〜23℃        11,787    36.3%
23〜27℃         9,157    28.2%
> 27℃（緊急）   9,448    29.1% ← 問題
─────────────────────────
合計            31,853   100.0%
```

**過剰発報率: 29.1%（全測定値の）**

→ Goodhart's Law（1975）の典型例:「閾値設定が緊急指標の有効性を喪失させた」

---

## ■ Anbai Curve（Figure 2）

本研究が提唱する「Anbai仮説」:

> 成果品質 Q は最適化投入量 I の単調増加関数ではなく、Anbai Point (I*) を超えると逓減し、Diminishing Point (I**) を超えると崩壊する。

```
Q
↑          ╭── Anbai Zone ──╮
│        ╱                   ╲
│      ╱                       ╲
│    ╱         最適域            ╲___
│  ╱
└────────────────────────────────→ I
         I*(最適)           I**(崩壊)
```

**初期システムはI**を超えていた**（I = 1.0回/時のLLM呼び出し）

---

## ■ 介入：三層アーキテクチャ

| Layer | 名称 | 技術 | 対象頻度 |
|-------|------|------|---------|
| 1 | 緊急停止 | if文 + 即時通知 | 稀（絶対条件） |
| 2 | 定常制御 | cron + ルールベース | **~99%** |
| 3 | 高度判断 | LLM | ~1% |

**制御設定ファイル**（Layer 2の核心）:
- 現場固有のルールを蓄積（詳細：NDA管理下）
- クライアントフィードバックにより継続的に更新
- AI技術者設計ではなく、現場経験の集積

---

## ■ 結果

| 指標 | 介入前 | 介入後 | 変化率 |
|------|--------|--------|--------|
| LLM呼び出し頻度 | 1.0回/時 | 0.11回/時 | **-89%** |
| 過剰発報率 | 29.1% | 改善（詳細NDA） | 大幅改善 |
| クライアント評価 | 問題あり | 改善 | 定性的改善 |

---

## ■ 考察

### Simon（1956）との対応

本結果は、satisficing の概念をAI制御システムに拡張した事例として解釈できる。「LLMを常時使用すること＝最大投資」が必ずしも最高品質をもたらさないことが実証された。

### 塩分アナロジー（直感的理解）

| 塩分濃度 | 感覚 | AI対応 |
|--------|------|--------|
| 0% | 無味 | 未使用 |
| 0.8% | 最適 | Anbai Zone |
| 2.0% | 過剰 | 毎時LLM |
| 5.0% | 崩壊 | AGI全振り |

---

## ■ 結論

1. LLM使用の89%削減と制御品質向上は両立可能であることが実証された
2. 過剰最適化（Over-Optimization）はGoodhart's Lawで説明可能
3. Simon（1956）の satisficing は現代LLMシステム設計に有効な指針を提供する
