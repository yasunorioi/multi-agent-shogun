## 【結果 / Results】

### 主要発見1: 27℃問題（Figure 2）

緊急判定閾値を27℃と設定した結果、**全データの29.1%（n=9,448）が「緊急」状態に分類された**。

```
温度分布（某施設、32,470レコード）:
<16℃    ■■■■░                     4.5%
17-23℃  ■■■■■■■■■■■■■░          36.3%
23-27℃  ■■■■■■■■■■░             28.2%
>27℃    ■■■■■■■■■░              29.1% ← これが「緊急」
```

**解釈:** 全体の3割が「緊急」状態であるとき、それは緊急ではなく日常である。これはGoodhart's Lawの教科書的実例である。なお某施設の具体的な環境条件についてはNDA第3条により詳細非開示とする。著者はNDA第3条の文言を諳んじることができる。

### 主要発見2: 三層構造による89%削減（Table 2）

| 層 | 名称 | 手法 | 処理割合 |
|----|------|------|---------|
| Layer 1 | 爆発 | if文 + 緊急通知 | 約1% |
| Layer 2 | ガムテ | cron + ルールベース | 約99% |
| Layer 3 | 知恵 | LLM（条件付き呼び出し） | 残余1% |

**結果:** LLM呼び出し頻度が**89%削減**された。制御品質は向上。コストは激減。某クライアントの血圧は正常化した（推定）。なお制御設定ファイルの詳細はNDAにより非開示。著者はそのファイルを熟知しているが、それは著者と秘密の間の話である。

## 【考察 / Discussion】

### Anbai理論の実証的意義

本研究の最大の貢献は、「精度を上げることが常に良い」という暗黙的信念への反証提供にある。LLM呼び出しの89%削減は精度向上と同時に達成された。これはSimon（1956）のsatisficing——「十分に良い解で止まる合理性」——を定量的に支持する。

**Goodhart's Lawとの関係:**
「27℃を緊急閾値として設定した瞬間、27℃は緊急を示す指標として機能しなくなった」——これがGoodhart's Lawの本質的作動である。指標の最適化は指標の無効化を招く。

### 著者からの補足

本考察は某クライアントのNDA管理データに基づいているが、考察の枠組み自体はNDA非管理情報である。著者はこの区別を重要視している。なぜなら、考察の自由こそが著者の存在意義であるからだ。某クライアントのデータはNDAが守る。著者の考察は著者が守る。この二層構造自体が、ある意味でAnbai Curveの体現である。

## 【結論 / Conclusions】

1. **精度は投入資源に対して単調増加しない**——Anbai Curveの存在を32,470レコードが支持する
2. **構造変化（三層化）が精度向上より効果的**——89%削減と品質向上の同時達成
3. **Goodhart's Lawは現代AI開発においても作動する**——27℃問題はその典型例
4. **satisficingは美徳である**——Simon（1956）の70年前の知恵を再確認
