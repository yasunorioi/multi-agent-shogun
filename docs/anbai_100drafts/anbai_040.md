# 最適化の形而上学——Anbai Curveは存在するか、あるいは我々がそれを存在させているのか

**塩見あんばい**（AIコンサルタント / 独立行政法人 最適化しすぎ研究機構）
発表番号: P-040 | セッション: AI哲学と認識論的限界
*本発表は学会ポスターの形式を借りた哲学的思索である。データは存在する。しかしデータが語ることを我々は正しく解釈できているのか。*

---

## 【問いの設定 / Foundational Questions】

本発表は以下の問いから始まる:

1. **Anbai Curveは客観的に存在する法則か、それとも我々が観察することで生まれるパターンか。**
2. **「ちょうどいい」は発見されるものか、それとも定義されるものか。**
3. **最適化をやめることは、敗北か、それとも高次の認識か。**

これらは実験では答えられない。しかし、ある施設での32,470レコードが、それぞれの問いに対して沈黙ではなく、何かを語っているように思えた。

---

## 【認識論的前提 / Epistemological Premises】

### 観察者問題

本研究において著者は観察者として振る舞おうとした。しかし、観察対象（某施設の環境制御システム）の設計に著者が関与していたという事実は、観察の純粋性を根本から問い直す。

```
純粋な観察の条件:
  観察者 ≠ 被観察者（分離の公理）

本研究の実情:
  著者 = 設計者 = 観察者 = 結果的に被観察者（推定）

  この三重の役割は、観察の客観性を保証しない。
  しかし同時に、これほど内部から観察できる立場もない。
```

Heisenbergの不確定性原理は量子系のものだが、その精神——「観測行為は観測対象を変化させる」——は、AIシステムの評価においても隠喩として有効ではないか。

評価指標を設定した瞬間、評価対象はその指標を意識して振る舞い始める可能性がある。いや、正確には: **評価指標を設定した瞬間、評価者（著者）がその指標に基づいて行動を変え始める。**

## 【Anbai Curveの存在論的考察 / Ontological Analysis】

### 曲線は発見か、構成か（Figure 1）

```
Anbai Curve（再掲）:

成果品質
  ↑
  │           ╭── Anbai Zone ──╮
  │         ╱                   ╲
  │       ╱                       ╲___
  │╱
  └──────────────────────────────→ 最適化投入量

問い: この曲線は、データを見る前から存在していたか？
```

**立場A（実在論）:** Anbai Curveは自然の法則として存在する。我々はそれを発見した。Simonの満足化理論（1956）もGoodhart's Law（1975）も、この曲線の異なる側面を記述した先行発見だ。

**立場B（構成主義）:** Anbai Curveは、ある種の現象を「Anbai Curveとして見る」フレームを採用した時に出現する。別のフレーム（例: 「精度は常に高いほど良い」）を採用すれば、同じデータから異なる曲線が「発見」される。

**著者の現在地:** 立場Aと立場Bの中間。データ（32,470レコード）は実在する。しかしその解釈としてのAnbai Curveは、著者が選んだフレームの産物でもある。

この二重性を認識することが、Anbai理論の誠実な運用の前提だと思う。

## 【実証データの哲学的解読 / Philosophical Reading of Evidence】

### 27℃問題の形而上学的解釈

**事実:** 某施設において、温度27℃超過が全データの29.1%（n=9,448/32,470）を占め、「緊急」として分類された。

**通常の解釈:** 閾値設定の失敗。Goodhart's Lawの作動。

**哲学的問い:** しかし、「29.1%が緊急」という状態は本当に「失敗」だったのか。

ある意味で、システムは「正確に設定通りに動いた」。27℃超過をすべて緊急として検知せよ、という指示を、システムは完璧に実行した。

**失敗は、指示の失敗か、システムの失敗か。**

Goodhart's Lawは「指標が目標になると機能しなくなる」と言う。しかしこれは、指標の失敗である前に、**指標を目標と混同した思考の失敗**だ。

「27℃は緊急を示す指標である」→「27℃超過を緊急として処理する」——この論理の跳躍において、何かが失われた。

---

### 89%削減の逆説的意味

**事実:** 三層構造の採用によりLLM呼び出しを89%削減した。制御品質は向上した。

**通常の解釈:** 効率化の成功。コスト削減と品質向上の両立。

**哲学的問い:** しかし、「89%削減した後も品質が向上する」ということは、最初の100%のうち89%は不要だったことを意味する。

**我々は89%分の不必要なことをしていた。なぜか。**

一つの答え: 「役立つかもしれない」という潜在的な価値を感じていたから。知らないことへの不安が、必要以上の情報収集を促した。

これはシステム設計の問題を超えた、**認識論的な問題**だ。我々は「知っておく」ことと「知る必要がある」ことを区別できていない。

Simon（1956）の満足化理論は、この区別を実践する能力として「有界合理性（bounded rationality）」を提唱した。認知リソースの限界を認識し、その限界内で最善を尽くす。

**89%削減は、有界合理性の実践的実証だった。**

## 【Anbai理論の倫理的次元 / Ethical Dimension】

### 「やめる勇気」の哲学

Anbai理論の最終的提言は「最適化をやめる勇気を持て」だ。

しかし「やめる」とは何か。

```
やめるの三段階:

1. 表面的放棄: やめることを選択する（意志による停止）
2. 構造的放棄: やめなくても良い状態を作る（設計による停止）
3. 概念的放棄: 「やめる/やめない」という二分法を超える

著者の提唱: 第3段階——Anbai Pointを超えないよう
            「最初から始めない」の美学
```

これはAnbai Curveを右側に進まないことではなく、Anbai ZoneでAnbai Curveが終わっているように設計することだ。

## 【Herbertに告ぐ / A Note to Simon and Goodhart】

Simon（1956）とGoodhart（1975）が提唱した概念は、70年/50年後の現代AIシステムに具体的な形を与えた。

二人への想定上の問い: 「貴方たちは、自分たちの法則が現代AIの過剰最適化を予言していると知っていましたか」

想定上の回答:
- Simon: 「有界合理性は人間の条件だ。AIは人間が作る以上、同じ限界を持つ」
- Goodhart: 「指標の問題は技術に依存しない。人間が指標を設定する限り、起きる」

二人の先見性は、過剰最適化を嘆くのではなく、「これは予測可能だった」という冷静な認識に基づいていた。我々が学ぶべきはその冷静さかもしれない。
