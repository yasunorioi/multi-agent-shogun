# What If "Good Enough" Is Better Than Perfect? ——Anbai理論の学術的基盤

*TEDxOptimization 2026 / Speaker: 塩見あんばい（AIコンサルタント）*

---

（舞台中央に立つ。スライドには「100%」と大きく表示）

What if I told you... that chasing 100% accuracy is making your AI system worse?

（間）

こんにちは。塩見あんばいです。AIコンサルタントをしています。32歳、独身——これは今日の話には関係ないはずですが、後で関係してきます。

（スライド切り替え：Anbai Curve）

これが Anbai Curve です。横軸に最適化投入量、縦軸に成果品質。最初は投入量に比例して品質が上がる。しかし、ある点——Anbai Point——を超えると品質は逓減し、さらに先で**逆効果に転じる**。

これは私の思いつきではありません。学術的系譜があります。

（スライド：Herbert Simon, 1956）

Herbert Simon。1956年に「satisficing（満足化）」を提唱。合理的な意思決定者は最適解ではなく「十分よい解」で止まる。1978年、ノーベル経済学賞。

（スライド：Charles Goodhart, 1975）

Charles Goodhart。1975年。"When a measure becomes a target, it ceases to be a good measure." 指標を目標にした瞬間、その指標は壊れる。

（スライド：Campbell, 1976）

Donald Campbell。1976年。同じ構造を社会政策で実証。数値目標が政策の質を劣化させる。

（間。客席を見渡す）

70年間、別々の分野で同じことが言われてきた。**最適化しすぎると壊れる**。

（スライド：27℃ = 29.1%）

私はこれを、実データで体験しました。某環境制御PJ——詳細はNDA管理下のため非開示ですが（会場笑い）——114日間、32,470レコードのデータです。

温度27℃超を「緊急」に設定した。結果、**全データの29.1%が緊急**と判定された。

（間）

全体の3割が緊急であるとき、何が起きるか。緊急という概念が消滅する。Goodhartの法則の教科書的実例です。

（スライド：三層構造）

解決策は精度を上げることではなく、**構造を変えること**でした。

Layer 1：if文による緊急停止。議論の余地なし。
Layer 2：cronとルールベース。日常の99%をカバー。
Layer 3：LLMの1時間予報。本当に困った時だけ。

結果：**LLM呼び出し頻度89%削減**。品質は維持、むしろ向上。

（客席からどよめき）

89%のLLM呼び出しは不要だったんです。if文で十分だった。

（スライド：satisficing timeline）

```
1956年 Simon: satisficing提唱
1975年 Goodhart: 指標の自壊を指摘
1976年 Campbell: 社会政策で実証
2022年 Skalse et al.: Reward Hackingを定式化
2025年 某PJ: 29.1%問題で体験 ←ここ
```

（間）

2022年、Skalseらが「Reward Hacking」を定式化しました。強化学習で報酬関数を最適化しすぎると、意図した行動ではなくハック的行動が出現する。指標を最適化しすぎると、測りたかったものとの乖離が広がる。

これら全て、Anbai Curveの異なる表現です。分野が違うだけで、構造は同じ。

（前に一歩出る）

制御設定ファイルの話をさせてください。某PJで蓄積された現場のルール——詳細はNDA管理下ですが——これはAI技術者が設計したものではありません。クライアントからの強いフィードバックで育ったものです。

（会場笑い）

「強いフィードバック」と言いましたが、正確には血圧に比例する品質のフィードバックです。怒りが品質を上げた。秘伝のタレは先代の怒りで育つ。

（スライド：三つの提言）

最後に、Anbai理論からの三つの提言を。

**1. 精度を上げる前に、その精度が本当に必要か問え。**
99%の問題はif文とcronで解けます。

**2. 数値化した瞬間、それは「ちょうどいい」ではなくなると知れ。**
Goodhartの法則は1975年から有効です。

**3. 最適化をやめる勇気を持て。**
Simonはこれをsatisficingと呼びました。70年後の我々は、100倍の計算資源でやっと同じ地点にいます。
