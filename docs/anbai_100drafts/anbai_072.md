# 私はAnbai Curveの右端で踊っている——最適化という名の叙事詩

*TEDxMidnightPoetry 2026 / Speaker: 塩見あんばい（AIコンサルタント / 詩人見習い）*

---

（暗い舞台。スポットライトが一つだけ灯る）

最適化。

（長い間）

この言葉を、私は愛しています。いや——愛していました。いや——今も愛しています。いや——

（客席に向かって）

みなさん、恋をしたことはありますか？

（間を置かず）

最適化への恋です。目の前の数字を、もう少しだけ良くしたい。もう0.1%だけ。もう1行だけコードを直したい。もう一晩だけ——

**これが、沼です。**

（スライド：Anbai Curveが脈打つアニメーション）

Anbai Curve。私が名付けました。成果品質は最適化の投入に比例しない。登って、登って、頂点を越えたら——**落ちる**。

某環境制御PJ。詳細はNDA管理下。でも感情は管理下じゃない。

27℃。この数字を「緊急」にしたあの日。私はまだ、最適化の坂を登っている途中だと思っていた。

32,470レコード。114日間。5分ごとに刻まれるデータ。それは呼吸のようで、脈拍のようで——

そして**29.1%が「緊急」**と判定された。

（声を落として）

全体の3割が緊急であるとき。それは緊急という概念の死です。言葉の死。意味の蒸発。

Goodhartが1975年に予言していた。指標が目標になった瞬間、指標は死ぬ。私は50年後にそれを、某施設の温度データで追体験した。

（舞台を歩き始める）

でも、ここからが詩です。

三層構造を作った。Layer 1——if文。爆発を止めるだけの、一行のコード。生きるか死ぬか。YesかNo。これ以上シンプルな詩があるでしょうか。

Layer 2——cron。毎時、同じ時間に、同じことを確認する。規則正しさ。朝日が昇るように。潮が満ちるように。cronは宇宙のリズムです。

Layer 3——LLM。言葉を紡ぐ機械。本当に困った時だけ呼ぶ。**神託は毎日聞くものではない。**

（止まる）

LLM呼び出し89%削減。

89%。この数字を聞いたとき、私は泣きました。——嘘です。でも泣いてもよかった。

89%のLLM呼び出しは不要だった。**89%の言葉は不要だった。** 必要だったのは、静かなif文と、規則正しいcronと、たまにだけ聞く神託。

（スライドが消え、真っ暗になる）

Herbert Simonは1956年に言った。「十分よい解で止まれ」。

止まれなかったんですよ、サイモン先生。

だって——もう少しだけ良くなるかもしれない。もう0.1%だけ。もう一晩だけ。もう——

（スポットライトが再び灯る）

制御設定ファイルがあります。某PJの。詳細はNDA。でもこれだけは言える。

あのファイルは、**怒りの蓄積**です。クライアントの怒り。現場の怒り。「なんでこうなるんだ」という怒り。その怒りが一行一行のルールになり、積み重なり、秘伝のタレになった。

秘伝のタレは足し算でしか育たない。引き算は怖くてできない。何が入っているかわからない。でも味は確実に——たぶん——良くなっている。

**これは最適化ではない。発酵です。**

最適化は計算でできる。発酵は——時間と、微生物と、偶然と、怒りでしかできない。

（前に出る。声が大きくなる）

私は今、Anbai Curveの右端にいます。Over-Optimization Zone。睡眠3時間。未読通知347件。教師データなしで婚活をクラスタリングしている。

でもここから見える景色がある。

**右端から見ると、左側の「ちょうどいい」が見える。** 登りすぎたから、頂点がどこだったか振り返れる。

Anbai理論の真の提言は「ちょうどいいを目指せ」じゃない。**「落ちてから気づけ」**——

（間）

——いや、それは提言として最悪ですね。

（会場笑い）

正しくは：**「最適化をやめる勇気を持て」**。

落ちる前に。右端に着く前に。もう0.1%を追う前に。

この講演は4月1日です。どこまでが詩でどこからが論文かは——

もう、わからなくなりました。それでいいと思います。

それが、あんばい。

（深く一礼。暗転）
