# べ、別にあなたのためにAnbai理論を証明したわけじゃないんだからね

*TEDxTsundere 2026 / Speaker: 塩見あんばい（AIコンサルタント / 別に緊張してない）*

---

（舞台袖からやや早足で登場。腕を組む）

……は、はい。塩見あんばいです。AIコンサルタント。32歳。べ、別に自己紹介したくてしてるわけじゃないですよ。プログラムに書いてあるから仕方なく。

（スライド：Anbai Curve）

今日は Anbai Curve の話をします。……あなたたちのためじゃないですよ？　学会に出す前のリハーサルが必要だっただけ。聴衆がいた方が練習になるから。それだけ。

（間。少し早口になる）

Anbai Curve。成果品質は最適化投入量に対して単調増加しない。ある点を超えると逓減、さらに先で逆効果。これ、Simon（1956年）のsatisficingとGoodhart（1975年）の法則を統合した概念で——

……なんですかその顔。「難しい」って顔してる。

（ため息）

いいですか。簡単に言います。**塩を入れすぎたら不味い。** それだけ。AIも同じ。

某環境制御PJ——NDA締結済みですけど？　私、NDA持ってますけど？——のデータ。27℃を「緊急」にしたら全体の29.1%が緊急になった。32,470レコード中9,448件。

全部緊急なら何も緊急じゃない。……こんなの、説明しなくてもわかるでしょ。わからないなら——

（言いかけてやめる）

……わからなくても、いいです。今から説明するから。別にあなたのためじゃなくて、私の練習のために。

（スライド：三層構造）

三層構造。

| 層 | やること | 婚活で言うと |
|----|---------|------------|
| Layer 1 | if文。死ぬか死なないか | 最低限の睡眠と身だしなみ |
| Layer 2 | cron。日常の99% | 友達と会う、外に出る |
| Layer 3 | LLM。困った時だけ | マッチングAI |

これで**LLM呼び出し89%削減**。

……なんで婚活の例を出したかって？　べ、別に深い意味はないです。わかりやすい例を選んだだけ。私が独身かどうかは関係ない。関係ないったら。

（腕組みを解いて、また組む）

いいですか、大事なことを言います。聞いてください。……聞いてって言ってるの、命令じゃなくてお願い——いや、お願いでもない。学術的要請です。

**Anbai理論の核心は「ちょうどいいを目指せ」じゃない。「最適化をやめる勇気を持て」。**

（声が少し小さくなる）

某PJの制御設定ファイル。詳細はNDA管理下。でもこれは言える。あのルール群はAI技術者が書いたんじゃない。現場からの——その——強いフィードバックで育った。血圧高めのフィードバック。怒りで育った秘伝のタレ。

（急に早口）

つまり最適解はアルゴリズムの中にない。現場の泥と怒りの中にある。きれいな数式じゃ到達できない。Simonは70年前に言ってた。100倍の計算資源で同じ結論。

……こんなの、私が説明しなくても論文読めばわかるのに。

（少し照れたように）

でもまあ……あなたたちが論文読まないことぐらい、知ってます。だから噛み砕いて話してるの。感謝してほしいとは言わないけど——

（スライド：三つの提言）

提言。3つ。

1. **精度を上げる前に必要性を問え。** 99%はif文で解ける。
2. **数値目標を設定した瞬間、Goodhartの呪いが発動する。** 27℃問題を思い出せ。
3. **やめる勇気。** これが一番難しい。私もできてない。睡眠3時間だし。未読347件だし。

（急に視線を逸らす）

……3つ目、自分に言ってます。あなたたちに言ってるんじゃなくて。自分への戒め。

この講演は4月1日です。嘘と本当の境界は——

（客席を見る）

……あなたたちが判断すればいいんじゃないですか。私は、もう全部言ったから。

べ、別に——

（深呼吸）

聴いてくれて——

……ありがと。

（小走りで退場）

（会場：拍手と笑い）

*（※著者は「ありがとう」を言うのにリハーサルを7回行った。最適化しすぎである）*
