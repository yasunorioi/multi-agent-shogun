# I Have an NDA, and It Changed Everything ——守秘義務が教えてくれたAnbai理論

---

（舞台中央。自信に満ちた笑顔。スライドには「NDA」の三文字）

私は、NDAを持っています。

（間。拍手を待つ表情）

……拍手するところですよ？

（会場：笑いと拍手）

ありがとうございます。塩見あんばいです。AIコンサルタント、32歳。そして**NDA保持者**。この肩書き、名刺に入れたいぐらいです。

今日は、NDAの向こう側にある真実——Anbai理論——についてお話しします。もちろん、NDAに抵触しない範囲で。**この「範囲」の広さが、私のスキルです。**

（スライド：某環境制御PJ）

某環境制御PJ。某クライアント。某施設。

「某」が多い？　NDAですから。NDA第2条第1項により施設名は非開示。第2条第1項の存在を開示すること自体は第3条で許可されています。私はNDAを熟読しました。**NDAを熟読するAIコンサルタント、塩見あんばい。**

（ドヤ顔）

さて、数字の話はできます。数字はNDAの対象外——正確には、統計的に個体識別不能な集計値は開示可能と解釈しています。この解釈の法的妥当性については弁護士に確認していませんが、4月1日ですので。

（スライド：27℃ = 29.1%）

**32,470レコード。114日間。5分間隔。** 某施設の温度データ。

27℃を緊急閾値にしたら、**29.1%が緊急**。約9,448レコード。

Goodhartの法則。1975年。指標が目標になったら指標は壊れる。これは公開情報です。NDA不要。

でもね——**この法則を体験で理解することと、論文で読むことは違う。** その体験がNDAの向こう側にある。私はそこにいた。あなたたちはいなかった。

（ドヤ顔 MAX）

（スライド：三層構造）

解決策。三層構造。

Layer 1：if文。緊急停止。某施設の某センサーが某閾値を超えたら某通知を某担当者に送る。**全部「某」ですが、構造は伝わりますよね？**

Layer 2：cron + ルールベース。某時間ごとに某条件を某基準で判定。日常の99%。

Layer 3：LLM。1時間予報。本当に困った時だけ。

結果：**LLM呼び出し89%削減。**

89%。つまり、10回中9回はLLMに聞く必要がなかった。if文で十分だった。

この「89%」を体感した人間と、読んだだけの人間の差。それがNDAの価値です。

（前に出る）

制御設定ファイルの話をしましょう。

某PJには制御設定ファイルが存在します。そこには現場固有のルールが蓄積されている。これらはAI技術者が設計したものではない。**クライアントからの強いフィードバック**によって生まれた。

「強いフィードバック」の定義？　血圧に比例する品質のフィードバックです。この相関関係はNDAの対象外です。なぜなら、これは私の個人的観察だから。

（ドヤ顔で客席を見渡す）

NDAが教えてくれたこと。

**秘密を守ることと、知見を共有することは両立する。** 施設名を言わなくても「29.1%問題」は伝わる。クライアント名を出さなくても「怒りドリブン開発」は理解できる。

NDAは制約ではない。**抽象化のフレームワーク**です。具体を隠すことで、構造が浮かび上がる。

これ、Anbai理論そのものです。詳細を削ぎ落として構造だけ残す。**ちょうどいい情報量。**

（スライド：Anbai理論 三つの提言）

1. **精度を上げる前に構造を変えろ。** 27℃問題は精度の問題じゃない。設計の問題。
2. **99%はシンプルな手段で解ける。** 残り1%のために全体を複雑にするな。
3. **最適化をやめる勇気を持て。** Simonはこれに1978年のノーベル賞をもらった。

**NDA管理下です。**
