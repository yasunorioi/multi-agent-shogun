# 29.1% ——この数字だけ覚えて帰ってください

*TEDxNumbers 2026 / Speaker: 塩見あんばい（AIコンサルタント / 数字の暴力装置）*

---

（舞台。スライドには巨大な「29.1%」）

29.1%。

（5秒の沈黙）

この数字だけ覚えて帰ってください。

塩見あんばいです。AIコンサルタント。今日は数字の話しかしません。形容詞は使いません。感情はバイアスです。

（スライド：データセット概要）

| 項目 | 値 |
|------|-----|
| 期間 | 114日間 |
| レコード数 | 32,470 |
| 間隔 | 5分 |
| 情報源 | 某環境制御PJ（NDA管理下） |

某施設の温度データ。「某」はNDAによる制約であり、データの信頼性には影響しません。

（スライド：温度分布）

| 温度帯 | 比率 | レコード数 |
|--------|------|-----------|
| <16℃ | 4.5% | 1,461 |
| 17-23℃ | 36.3% | 11,787 |
| 23-27℃ | 28.2% | 9,157 |
| >27℃ | **29.1%** | **9,448** |

設計者（私です）は27℃を「緊急」閾値に設定。結果、3割が緊急。

数字で言います。**29.1%の事象は「稀」ではない。** 二項分布で考えれば、1日288レコード（24h×12回/h）中約84件が緊急。1時間に約3.5件。17分に1回の緊急。

17分に1回の緊急を処理するシステム。それはもう緊急対応ではなく日常業務です。

（スライド：Goodhartの数式）

Goodhart（1975年）を数式で表現します。

```
指標の有用性 = f(目標からの独立性)
目標として設定 → 独立性 = 0 → 有用性 = 0
```

27℃を目標にした瞬間、27℃の情報量（Shannon entropy的に言えば）が最小化された。

（スライド：三層構造のコスト分析）

三層構造導入後の数字。

| 層 | 処理比率 | コスト比率 | 精度 |
|----|---------|-----------|------|
| Layer 1（if文） | 0.1% | ~0% | 100%（二値判定） |
| Layer 2（cron） | 99.0% | ~5% | 95%以上 |
| Layer 3（LLM） | 0.9% | ~95% → ~11% | 可変 |

**全体のLLM呼び出し：89%削減。**

コスト構造の逆転。導入前はLayer 3が100%のコストを占めていた。導入後はLayer 2が処理量の99%を占め、コストは5%。

パレートの法則（80/20）を超えている。**99/5**。99%の処理を5%のコストで。

（前に出る。数字を指で刻みながら）

Simonのsatisficing（1956年）を数値化する試み。

```
最適解の探索コスト: C_opt
十分解の探索コスト: C_sat
品質差: ΔQ = Q_opt - Q_sat

C_opt >> C_sat のとき、ΔQ → 0
```

最適解を見つけるコストは十分解の数十倍かかるが、品質差はゼロに近づく。**投資対効果（ROI）は十分解で最大化される。**

某PJの実データで検証：Layer 2（ルールベース）の判定精度とLayer 3（LLM）の判定精度の差は、測定可能な範囲で有意差なし。95%の精度で十分だった。
