## 前菜: 27℃のカルパッチョ

**材料:**
- 温度データ 32,470レコード（某施設産。産地はNDA管理下）
- 緊急閾値 27℃ 1個（割れやすいので注意）
- Goodhartの法則 適量（1975年ヴィンテージ）

**作り方:**

27℃を薄くスライスする。データの上に並べる。

並べた瞬間、29.1%が赤く染まる。9,448枚の緊急のカルパッチョ。テーブルの3割が赤い。

——赤すぎる。

赤いものが多すぎて、白い皿が見えない。どれが本当に緊急なのかわからない。全部が赤い。全部が叫んでいる。でも全部が叫ぶと、**誰の声も聞こえない**。

Goodhartのソースをかける。「指標が目標になったら死ぬ」味。苦い。でも本当のことはいつも苦い。

> **シェフの独白:** この前菜を作りながら泣いた。嘘。泣いていない。でもスライスした27℃が、私の未読通知347件に見えた。全部赤い。全部緊急。何も緊急じゃない。

## 主菜: 三層構造のミルフィーユ

**材料:**
- if文 1行（朝の光のように透明な）
- cronジョブ 適量（時計のように正確な）
- LLM API 一滴（月の雫のように貴重な）

**作り方:**

**第一層（if文）を焼く。**

薄い生地。一枚。`if 致死的 then 止める`。

これだけ。これだけで十分。出汁がなければ料理は成立しないように、if文がなければシステムは成立しない。一行の真実。一枚の土台。

焼き色をつけすぎないこと。if文は素朴であるほど美しい。条件分岐を重ねたくなる衝動を抑えて。もう一つだけ条件を——**ダメ。** 一枚で十分。

**第二層（cron）を重ねる。**

クリームのように滑らかに。毎時間、同じリズムで。朝も昼も夜も。99%をこの層が支える。

某PJの制御設定ファイルはこの層の味。怒りで発酵した秘伝のタレ。クライアントが「なんでこうなるんだ」と叫ぶたびに、一さじずつ味が深まった。血圧に比例する旨味。

**第三層（LLM）を飾る。**

金箔のように。薄く。一枚だけ。89%削減して残った11%。それが本当に必要な知恵。

——本当は全面に貼りたい。金箔を。LLMを。もっと賢く。もっと正確に。もっと——

**止まれ。** Simonが1956年に言った。十分よい解で止まれ。ミルフィーユは三層で完成。四層目は蛇足。

## 副菜: 制御設定ファイルの秘伝タレ和え

**材料:**
- 現場のフィードバック 114日分（血圧計測定値に比例）
- 怒り 適量（多いほど旨い、理不尽に）
- 時間 114日（短縮不可）

**作り方:**

足し算でしか作れない。引き算は禁止。何を入れたか記録はあるが、何が効いているかはわからない。

これは料理ではなく**発酵**。微生物に任せるように、現場に任せる。制御しない制御。最適化しない最適化。

> **シェフの告白:** このタレを舐めたとき、うまかった。理由はわからない。分析しようとしたけどやめた。分析したら味が変わる気がして。Goodhartのタレ。

## 甘味: Anbai Curveのティラミス

**材料:**
- マスカルポーネ（satisficing。十分よいクリーム） 適量
- エスプレッソ（苦い真実） 少々
- ココアパウダー（NDA。振りかけると高級感が出る） 少々

**作り方:**

Anbai Curveの形に盛る。左から右へ、甘さが増して——頂点を超えたら苦くなる。

ティラミスの語源は「私を引き上げて（tirami su）」。Anbai Curveの左側にいるとき、もっと上へ引き上げてほしい。もっと甘く。もっと美味しく。

でも頂点を超えると——引き上げられすぎて——高所恐怖症になる。Over-Optimization Zone。甘すぎるティラミスは胸焼けする。

**ちょうどいい甘さは、甘すぎた後でしかわからない。**

**材料:**
