## 下ごしらえ

27℃を閾値に設定する。

……**しないでください。** 設定したら29.1%が緊急になるから。某PJで実証済み。114日間。NDA管理下だけど、あなたが同じ失敗するのを黙って見てるわけにはいかないから言ってるの。べ、別に——

Goodhartの法則（1975年）。指標を目標にしたら壊れる。知ってる？ 知らないの？ ……しょうがないから教えてあげる。

塩分で言うと。0.8%がちょうどいい。2.0%でしょっぱい。5.0%で食べられない。27℃閾値は塩分5.0%。しょっぱすぎ。

**だから先に味見してから塩を入れなさいって言ってるの。** ……言ってなかったかもしれないけど、今言ったから。

## 調理手順

### Step 1: Layer 1（if文）

if文を1行書く。

```
if 致命的 then 停止
```

はい、これだけ。……何、もっと複雑にしたいの？ しなくていい。1行で十分。Simon（1956年）も言ってる。satisficing。十分よければいいの。ノーベル賞の人がそう言ってるんだから素直に従いなさい。

**調理のコツ:** if文に3つ以上の条件を入れたくなったら、一度キッチンを出て深呼吸しなさい。……べ、別に心配してるんじゃなくて、if文が壊れると直すのが面倒だから。

### Step 2: Layer 2（cron）

cronを回す。毎時間。日常の99%を処理する。

某PJの制御設定ファイル——NDA管理下だけど構造は教えてあげる——は現場のフィードバックで育った。怒りで育った秘伝のタレ。血圧に比例する品質の出汁。

あなたにもそういうフィードバックをくれる人がいる？ いるなら大事にしなさい。怒ってくれる人は貴重よ。……べ、別に羨ましくなんかないけど。

### Step 3: Layer 3（LLM）

LLMは**一滴だけ**。

89%削減。某PJで10回中9回はLLMが不要だった。つまりあなたも、LLMを呼ぶ前にLayer 2で解決できないか考えなさい。

……LLMに頼りすぎてるでしょ。わかるのよ、見てなくても。

## 味見

味見しすぎないこと。味見のたびに塩を足す人いるでしょ。**あなたのことよ。** Goodhartの法則。味見自体が味を変える。

「ちょうどいい」は感覚。数値化した瞬間に消える。だから教えられない。自分で見つけなさい。

……見つけ方のヒントぐらいは教えてあげる。**やりすぎたと感じた瞬間が、Anbai Pointを超えた瞬間。** そこから一歩戻る。
