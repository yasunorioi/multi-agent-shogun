# 『新卒エンジニアのためのAnbai定食』——最初に覚えるべき三品

*塩見あんばいの最適化キッチン vol.7 〜新人研修編〜*
*調理時間: 研修期間 / 難易度: ★☆☆☆☆（でも一番大事） / カロリー: 成長に必要な分だけ*

---

## 新卒のみなさんへ

社食のメニューみたいに書きます。わかりやすく。先輩として。

今日の定食は3品。これだけ覚えれば、エンジニアとして生きていける。

## 本日の定食メニュー

```
一の皿: if文の味噌汁 ——全ての基本
二の皿: cronの焼き魚 ——毎日食べても飽きない
三の皿: LLMの季節のデザート ——特別な日だけ
```

ご飯（データ）はセルフサービス。32,470レコード分ぐらい用意しておくと良い。

---

## 一の皿: if文の味噌汁

**材料:** if文 1行、出汁 適量

**作り方:**

```
if 危険 then 止める
```

以上。

新卒のみなさん。これ、研修で最初に書きますよね。`if (x > 0) { ... }`。これがエンジニアリングの味噌汁です。毎日飲む。飽きない。なくなると困る。

某環境制御PJ——先輩（私）はNDA持ってます、いつか持てるといいですね——で、このif文が最も重要な層（Layer 1）でした。

27℃を閾値にしたら29.1%が緊急になった話、知ってますか？ 知らない？ じゃあ教えます。全データの3割が「やばい」になったんです。テストで30人中9人不合格なら、テストが悪い。閾値が悪い。

**if文のコツ:** シンプルに保つ。条件を3つ以上入れない。味噌汁に具を入れすぎると味噌汁じゃなくなるのと同じ。

**コスト:** ほぼゼロ。味噌汁は安い。でも一番大事。

---

## 二の皿: cronの焼き魚

**材料:** cronジョブ 1個、ルール 適量

**作り方:**

cronを毎時間回す。定期的に同じことをチェックする。新卒のみなさん、cronって知ってますか？ 「毎時間○○を実行する」っていうスケジューラです。

焼き魚みたいなもの。毎日出てくる。地味。でも栄養がある。日常の99%はこれで回る。

某PJの制御設定ファイル——NDA管理下ですが——はこのcronの層で動いていました。現場のフィードバックで育ったルール。クライアントの怒りで味付けされた秘伝の焼き魚。

**新卒への教訓:** 先輩やクライアントに怒られたら、それはレシピへのフィードバック。怒りを受けて改善するたびに、味が良くなる。**怒られないエンジニアは味が薄い。**

ただし、パワハラはフィードバックじゃない。焼き魚を顔に投げるのは料理ではない。区別して。

**コスト:** 低額固定。焼き魚は高くない。

---

## 三の皿: LLMの季節のデザート

**材料:** LLM API 少々

**作り方:**

デザートは毎日食べない。特別な日だけ。

LLMは高い。トリュフのケーキみたいなもの。美味しいけど、毎食は無理。某PJでは89%削減した。10食中9食はif文の味噌汁とcronの焼き魚で十分だった。

**新卒がやりがちな間違い:** 「AIで解決しよう！」と最初からLayer 3（デザート）に飛びつく。味噌汁（if文）を飲んでから言え。焼き魚（cron）を食べてから言え。

Simon先生（1956年。ノーベル賞。すごい人）が言ってる。「十分よい解で止まれ」。味噌汁で十分なのに、トリュフケーキを注文するな。

**コスト:** 従量課金。食べた分だけ請求される。89%節約可能。

---

## 定食の食べ方（正しい順序）

1. **味噌汁（if文）** から飲む。基本を確認。
2. **焼き魚（cron）** を食べる。日常を回す。
3. **まだお腹空いてたら** デザート（LLM）を頼む。
4. **デザートを最初に食べるな。** 血糖値が乱高下する（コストが爆発する）。
