## 婚活の27℃問題

某PJで27℃を「緊急」にしたら、29.1%が緊急になった。32,470レコード中9,448件。全部緊急。何も緊急じゃない。

**婚活版:** マッチングアプリで条件を設定する。「年収○○万以上」「身長○○cm以上」。Goodhart（1975年）の法則。条件を目標にした瞬間、その条件で測りたかった「相性」は測れなくなる。

さらに「いいね」を全員に送ると、全員が候補。未読通知347件。27℃問題の恋愛版。

**レシピの教訓:** 閾値を設定する前に、何を本当に測りたいのか考える。某PJでは27℃ではなく三層構造で解決した。婚活でも条件ではなく構造で考える。

## 正しい婚活レシピ（三層構造）

### 一の皿: Layer 1のスープ（まず生きろ）

**材料:** 睡眠7時間、栄養、最低限の身だしなみ

某PJのif文。生きるか死ぬか。これを削ったら全部終わり。

**私の失敗:** 睡眠3時間。Layer 1削減。某PJで「if文（Layer 1）を削るな、削ったら制御崩壊」と実証した人間が、自分の睡眠を削っている。

**レシピの鉄則:** スープ（生存基盤）を抜いた定食は成立しない。どんなに良いメイン料理（Layer 3）があっても。

### 二の皿: Layer 2のメイン（人と会え）

**材料:** 週1回の友人との食事、月2回の新しい場所

cronの焼き魚。毎週、定期的に。地味だけど99%の栄養はここから。

某PJの制御設定ファイル。怒りのフィードバックで育った秘伝のタレ。NDA管理下だけど教訓は明白：**現場に出ないとデータは取れない。** レシピを読むだけでは味はわからない。

**私の失敗:** Layer 2スキップ中。人と会っていない。教師データが取れない。**マッチングアプリの設定（Layer 3）を30分いじるより、30分散歩に出た方が出会いの確率は高い。**

### 三の皿: Layer 3のデザート（たまにだけ）

**材料:** マッチングAI 少々

LLMは困った時だけ。89%削減。婚活でも、AI分析は週末の趣味程度に。

**私の失敗:** Layer 3全振り。教師データなしで教師なし学習。クラスタリングしかできない。「この人がいい」とは言えない。会ってないから。
