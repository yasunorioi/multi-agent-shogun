**フィードバック品質の定量化試行:**

```
フィードバック品質 ∝ 発信者の血圧
ルール追加頻度 ∝ フィードバック品質
∴ ルール追加頻度 ∝ 血圧
```

これは厳密な数式ではなく、某PJの観察に基づく経験則。しかし相関は観測された（N=小、p値=未計算）。

## Anbai Curve の数式表現

```
Q(x) = a·x·exp(-b·x)  (x > 0, a > 0, b > 0)

Q: 成果品質
x: 最適化投入量
a: 初期効率係数
b: 逓減係数

dQ/dx = a·exp(-b·x)·(1 - b·x)
dQ/dx = 0 のとき x* = 1/b（Anbai Point）

x < 1/b: Under-Optimization（投入不足）
x = 1/b: Anbai Point（最適）
x > 1/b: Over-Optimization（過剰投入）
```

某PJの実データでb値を推定する十分な情報はNDA管理下。しかし、曲線の形状が上記モデルと整合することは29.1%→89%削減のデータから示唆される。

## Satisficing の数値的正当性

Simon（1956年）を定量化する：

| 変数 | 定義 | 某PJ推定 |
|------|------|---------|
| C_opt | 最適解探索コスト | 基準=100% |
| C_sat | 十分解探索コスト | ~11%（89%削減後） |
| ΔQ | 品質差(最適-十分) | ≦0%（十分解の方が良い場合あり） |
| ROI_opt | 最適解のROI | Q/100% |
| ROI_sat | 十分解のROI | Q/11% ≈ 9×ROI_opt |

**十分解のROIは最適解の約9倍。** これがsatisficingの定量的根拠。
