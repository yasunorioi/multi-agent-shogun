## 2. LLM（大規模言語モデル）活用事例

### 2-1. 農研機構（NARO）— 国内初の農業特化型LLM

| 項目 | 詳細 |
|------|------|
| ベースモデル | Llama-3-ELYZA-JP-8B |
| 手法 | 継続事前学習 + QAファインチューニング + RAG + MoA（複数AI統合） |
| 学習データ | 1,000万トークン超（NARO研究データ・栽培暦・病害虫画像等） |
| 精度 | 汎用LLM（ChatGPT, Gemini）比で**正答率40%向上** |
| 実装 | 2024年10月〜三重県でイチゴ栽培を対象に試験運用 |
| 対象ユーザー | 普及指導員・農業生産者 |
| 狙い | 技術伝承の支援、指導員の負担軽減 |

> **殿のシステムとの比較**: NAROは8Bモデル+RAGで農業知識を補完。殿のuecs-llmはsystem_prompt.txt+tool_callの軽量アプローチで方向性は異なるが、「農業ドメイン知識をLLMに注入する」という課題は共通。

### 2-2. Nucleus AI — 農業OS構想（米国）

| 項目 | 詳細 |
|------|------|
| モデル | 22Bパラメータ LLM（MIT + 商用ライセンス） |
| 学習 | 1兆トークン（Web, Wikipedia, StackExchange, arXiv, コード） |
| ビジョン | AI駆動の「農業OS」— 需給最適化・不確実性の緩和 |
| 拠点 | カリフォルニア（Amazon, Samsung Research出身者） |

- 農業特化というより汎用LLMを農業OSの基盤にする構想
- 2023年にステルスから登場後、続報が限定的

### 2-3. AgriGPT — 農業ドメイン特化LLMエコシステム

| 項目 | 詳細 |
|------|------|
| 特徴 | Tri-RAGフレームワーク + マルチエージェントデータエンジン |
| 学習データ | Agri-342K（34.2万件の農業指示データセット） |
| 性能 | 汎用LLM比で農業タスクの適応性・推論力が大幅向上 |
| 論文 | arXiv:2508.08632（2025年） |

### 2-4. Greein（愛媛）— 地域特化型LLM

- e-kakashi独自ハイブリッドAI + LLMで自然言語質問応答
- 愛媛の栽培文化に根ざした知識ベースを活用
- **殿の「農家の怒り駆動開発」と親和性が高い**: 地域・畑ごとの知識蓄積型

## 3. 温室制御AIの事例（海外含む）

### 3-1. Autonomous Greenhouse Challenge（オランダ、WUR）

| 回 | 時期 | 概要 |
|----|------|------|
| 第4回 | 2024年秋 | 6つの高精度温室区画で国際チームが競争 |

- トマト栽培の純利益最大化（コスト vs 収量のバランス）が目標
- 気候・作物管理戦略をAIアルゴリズムで自律決定
- **世界で最も実践的な温室AI競技**: 実物の温室で複数チームが同時運用

### 3-2. 深層強化学習（DRL）による温室制御

| 研究 | 成果 |
|------|------|
| Adesanya et al. (2023) | DRL+PID制御で**エネルギー消費57%削減** |
| Sensors (2024) | IoT+RLエージェントによる温室温度制御（2023.11-2024.5実証） |
| Arxiv (2025) | 完全自動意思決定システムの課題と機会の包括レビュー |

- 最適温度: 昼間23-24℃（光合成酵素活性最大）、夜間16℃（呼吸損失最小化）
- **殿のPID+LLMハイブリッドは先進的**: DRL単体より三層構造の方が実運用に強い

### 3-3. BrainBox AI（カナダ）— 建物HVAC最適化

- 建物のHVACを自律制御 → エネルギーコスト25%削減、GHG排出40%削減
- クラウドベースで既存設備に後付け接続
- **温室への応用可能性**: 建物の空調制御と温室の換気制御は構造的に類似

### 3-4. 国内温室制御AI

| 企業 | 技術 |
|------|------|
| 誠和 | 「プロファインダー」環境制御システム |
| イノチオ | 統合環境制御盤 + データ分析 |
| デンソー | AI自動野菜収穫ロボ（24時間バッテリー稼働） |

- 国内は**ルールベース+閾値制御が主流**。LLMやDRLによる自律制御は研究段階
- 完全閉鎖型植物工場ではAI+LED制御で水使用量98%削減の事例あり

## 4. 農家のIT導入における課題・ギャップ

### 4-1. 三菱総研（2025年3月）の分析

[「日本のスマート農業はなぜ普及が進まないのか？」](https://www.mri.co.jp/knowledge/opinion/2025/202503_3.html)

| 課題 | 詳細 |
|------|------|
| **投資回収の不透明さ** | ロボットトラクタ1,000万円台。トマト収穫ロボは労働時間削減できたが、導入費含め合計収支マイナス |
| **経営規模の壁** | 大規模（20ha以上）は全体の1.6%。大多数の中小農家はROI合わない |
| **データ形式の不統一** | メーカー間でデータ互換性なし。囲い込み問題 |
| **通信インフラ** | 離島・中山間地域で5G/Wi-Fi未整備 |

### 4-2. 農家側の課題

| 課題 | 詳細 |
|------|------|
| **ITリテラシー** | 農業従事者の平均年齢68.4歳（2024年）。スマホ操作がギリギリの層が主力 |
| **月額コスト忌避** | ハードは設備投資として腹が決まるが、SaaS月額は永遠に気になる（← **殿と同じ心理**） |
| **「使いこなせない」不安** | ドローン操作・環境制御設定・データ分析に専門知識が必要 |
| **成功体験の不足** | 周囲に成功事例がないと踏み切れない |

### 4-3. 理想と現実のギャップ

```
【理想】                          【現実】
AI が最適制御してくれる    →    設定項目が多すぎて触れない
データで収量UP             →    データを見る時間がない
月数千円で使い放題         →    初期費用100万+月額数千円+保守
5Gで全圃場つながる        →    畑にWi-Fiすら届かない
若い人がIT使う             →    後継者がいない
```

### 4-4. 殿のアプローチが解決する課題

| 農家の課題 | 殿のuecs-llmでの対策 |
|-----------|---------------------|
| 月額コスト忌避 | NullClaw(ゼロコスト)デフォルト + LLM自然減衰モデル |
| 設定が難しい | system_prompt.txtに怒りを書けば制御ロジックになる（怒り駆動） |
| ITリテラシー不足 | LINE Bot「開けろ」「閉めろ」が最重要機能（稼働済み） |
| 高額な専用機器 | Pico(1,500円)+UniPi+RPi。マクガイバー精神 |
| オフライン不安 | 三層構造: Layer1/2はオフライン動作、Layer3のみAPI |
| データ互換性 | UECS/MQTT標準プロトコル準拠 |

## 5. 所感（調査者メモ）

- **LLMの農業活用は2024-2025年が転換点**: NARO、AgriGPT、Nucleusと相次いで農業特化LLMが登場
- **しかし温室制御にLLMを使う事例はほぼ皆無**: DRL/MPC/PIDが主流。殿のLLM+三層構造は独自性が高い
- **農家の課題の本質は「コストと使いやすさ」**: 技術の洗練度よりも「月額ゼロ」「LINE一言」が刺さる
- **殿のsystem_prompt.txt方式は農業LLMの新パラダイム**: モデル訓練ではなくプロンプトに農家の怒りを蓄積する。NARO方式(ファインチューニング)との対極にあるが、個別農家への適応速度は圧倒的に速い
