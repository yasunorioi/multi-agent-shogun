# ADR-0005: 高札FTS5全文検索

- **Status**: accepted
- **Date**: 2026-03-04

## Context

没日録DBへの完了タスク永続化（ADR-0001）により、過去のcmd/subtask/reportが蓄積された。しかし以下の問題があった：

1. **検索性の欠如**: SQLite CLIで直接クエリを叩く必要があり、足軽・お針子はDB権限なし
2. **知見の断絶**: 過去に同じ問題を解決した知見が活用されない
3. **家老の負荷**: コンパクション復帰時に過去タスクの文脈を手動で再構築する必要がある

## Decision

高札（kousatsu）にFTS5全文検索機能を持つ検索APIを実装する。

- **高札API**: Dockerコンテナ（ooku:agents.1）で稼働、`http://localhost:8080`
- **検索エンドポイント**: `GET /search?q=キーワード&limit=5` で没日録DBを横断検索
- **docs配信**: `GET /docs/context/ファイル名` でcontext/*.mdを配信
- **権限モデル**: 読み取り専用API。DB書き込みは引き続き家老のみ（ADR-0001の制約維持）
- **FTS5**: SQLiteのFTS5拡張を使用し、日本語全文検索に対応

代替案：
- **Elasticsearch/Meilisearch**: 追加インフラが必要でオーバースペックのため不採用
- **grep検索**: 構造化データの横断検索に不向きなため不採用
- **全エージェントにDB読み取り権限付与**: 権限分離の原則（ADR-0001）に反するため不採用

## Consequences

- **メリット**: 全エージェントが過去の知見を検索可能、家老のコンパクション復帰が高速化、重複作業の防止
- **デメリット**: Dockerコンテナの運用コスト、FTS5の日本語トークナイズ精度の限界
- **制約**: 高札がダウンしても業務は継続可能（高札は補助。検索なしでもタスク分解は実行可能）
- **将来**: ADR文書の検索統合、context/*.mdの検索統合を検討
