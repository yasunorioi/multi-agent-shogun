## §2 核心技術の解剖

### 2a. Volterra級数 + カーネルトリック

**本質**: 非線形システム `y(t) = f[t, u(·)]` をVolterra級数に展開すると無限個の係数が現れるが、カーネルトリック（Representer定理）により、データ数Nの次元の行列計算に帰着させる。

```
入力 u ∈ R^m
  ↓ 特徴写像 φ(u) ∈ R^∞（無限次元）
  ↓ カーネル関数 k(u,z) = φ(u)ᵀφ(z)
  ↓ 多項式カーネル: k(u,z) = (1 + uᵀz)^p
  ↓ 指数カーネル:   k(u,z) = exp(uᵀz)
  ↓
予測: y = k(u)ᵀ [K + λNI]⁻¹ yN
```

**重要**: 解はカーネル関数kを通してのみデータに依存。Volterra級数の無限個の係数はカーネル行列Kの中に「繰り込まれる」。

### 2b. Lasso（L1正則化）によるスパース化

目的関数のL2正則化項をL1に置き換え:

```
Ridge:  Ψ = Σ|y - ηᵀφ(u)|² + λ||η||²    ← L2: 全係数を小さく
Lasso:  Ψ = Σ|y - ηᵀφ(u)|² + λ||η||₁    ← L1: 多くの係数を厳密に0に
```

**殿の言葉に翻訳すると**: Lassoは「棍棒で殴れる変数」だけを自動選択する道具。殴っても効かない変数は係数が0になって消える。

### 2c. SLDS（切替線形動的システム）

```
状態空間モデル:
  x_{t+1} = f(x_t, u_t) + v_t    ← 状態遷移
  y_t     = g(x_t) + w_t          ← 観測

SLDSの拡張:
  モード変数 s_t が「どの線形モデルを使うか」を切替
  EM アルゴリズムで学習（E: 隠れ変数推定, M: モデルパラメータ更新）
  ディリクレ過程で最適モデル数を自動決定
```

**殿の温室制御との接続**: 温室の3層制御（爆発→ガムテ→知恵）もまさにSLDS。状態s_tが「通常運転/警報/緊急」を切替え、各モードで異なる制御則が動く。

## §4 Dexter + カーネル法の統合可能性

### 4a. 統合アーキテクチャ

```
Dexter (DCF分析・財務データ取得)
  ↓ 財務特徴量 (FCF, ROIC, debt/equity, growth...)
  +

  ↓ Lasso選択済み変数 (インフラ指標)
  +
獏 (baku.py) のリサーチ結果
  ↓ オルタナティブデータ (衛星画像, ODA, コンテナ)
  ↓
  ╔════════════════════════════════════╗
  ║ 統合判断エンジン                    ║
  ║   Input:  Dexter DCF + Lasso特徴量  ║
  ║   Model:  カーネルリッジ回帰        ║
  ║   Output: 統合スコア (投資判断)     ║
  ╚════════════════════════════════════╝
```

### 4b. 統合のレイヤー

| レイヤー | 担当 | 手法 | 殿の言葉 |
|---------|------|------|---------|
| L1: データ収集 | Dexter + 獏 | API/WebSearch | 弾を集める |
| L2: 変数選択 | Lasso | L1正則化 | 棍棒を選ぶ |
| L3: 非線形予測 | カーネル回帰 | RBF/多項式カーネル | 棍棒で殴る |
| L4: レジーム判定 | SLDS/HMM | EMアルゴリズム | 戦場を読む |
| L5: 意思決定 | 軍師 + GA適合度 | fit3 (ロス最小化) | 退却か突撃か |

### 4c. DexterのDCF特徴量へのカーネル展開

DexterのDCF SKILL.mdが出力する特徴量:
- FCF成長率（5年）、WACC、Terminal Value比率、EV/FCF倍率

これらを**カーネル回帰の入力特徴量として使える**:

```
DCF特徴量 u = [FCF_growth, WACC, TV_ratio, EV_FCF_multiple]
  ↓ 多項式カーネル k(u,z) = (1 + uᵀz)²
  ↓ 2次の交互作用を自動捕捉
  ↓ 例: FCF_growth × WACC の交互作用（成長率が高くてもWACCが高ければ割引）
```

**評価**: 理論的に可能だが、DCF分析自体がすでに非線形モデル。カーネル展開の追加価値は限定的。
**判定**: △ やる価値はあるが、L2（Lasso変数選択）の方がROI高い。

## §5 SLDS × 温室制御のクロスドメイン接続

### 5a. 構造的同型性

| 要素 | 温室制御 | systrade |
|------|---------|----------|
| **状態変数 x_t** | 温度, 湿度, CO2, 日射量 | 株価, 出来高, ボラティリティ |
| **入力 u_t** | 換気, 暖房, 灌水, CO2施用 | 買い, 売り, ホールド |
| **モード s_t** | 通常/警報/緊急 | 上昇トレンド/レンジ/暴落 |
| **観測 y_t** | センサー値 | ティックデータ |
| **切替条件** | 温度閾値, 異常検知 | ボラティリティ急変, VIX |

### 5b. 共通する数学構造

```
温室:  x_{t+1} = f_{s_t}(x_t, u_t) + v_t   where s_t ∈ {通常, 警報, 緊急}
市場:  x_{t+1} = f_{s_t}(x_t, u_t) + v_t   where s_t ∈ {トレンド, レンジ, 暴落}
```

**同じEMアルゴリズムで学習可能**。温室で蓄積した制御ログ（UECS-LLMのセンサーデータ）と、市場データは、SLDSの枠組みでは同じ問題構造を持つ。

### 5c. クロスドメインの実用価値

**高い**: 温室制御で培った「モード切替の検出 → 制御則の切替」は、市場レジーム検出にそのまま転用可能。特にGA適合度関数(fit3)の「トレンド/レンジ相場自動切替」は、SLDSのモード変数s_tそのもの。

**注意**: 温室は物理法則に従う（熱力学）。市場は人間の心理に従う。同じ数学構造でも、ノイズの質が根本的に異なる。温室のSN比 >> 市場のSN比。

## §6 2010→2026の技術発展と現在地

### 6a. カーネル法の歴史的位置づけ

```
2000年代: カーネル法の黄金期
  ├─ SVM/SVR が万能ツールとして君臨
  ├─ Volterra級数のカーネル化（本論文）
  └─ 「kernelize everything」の流れ

2012年: 深層学習の台頭（AlexNet）
  ├─ カーネル法の相対的地位低下
  ├─ NN が特徴抽出を自動化（手動カーネル設計不要）
  └─ ただし理論的理解はカーネル法の方が上

2020年代: LLM/Transformer時代
  ├─ 自己注意機構 ≒ カーネル関数の一般化
  │   （Attention(Q,K) ≈ softmax(QKᵀ) ← カーネル行列の変種）
  ├─ Random Feature Approximation がTransformerの効率化に使われる
  └─ カーネル法は「理論的バックボーン」として再評価

現在(2026): 実用的な使い分け
  ├─ 少量データ・解釈性重視 → カーネル法/Lasso（殿の戦場）
  ├─ 大量データ・精度重視 → 深層学習
  └─ テキスト/画像の理解 → LLM/Vision（Dexter/獏の領域）
```

### 6b. 殿の戦場での最適解

**殿のデータ量は「少量」に分類される**:
- アジアインフラ株の対象銘柄: 数十〜数百
- 地価データ: 年次、数百〜数千サンプル
- ODA/ADB案件: 年間数十件

→ **深層学習は過剰。カーネル法/Lassoが最適解。** これは2010年の論文の技法がそのまま使えることを意味する。

### 6c. 2010年論文で言及されなかった2026年の進展

| 技術 | 2010年の状況 | 2026年の現在 |
|------|-----------|-----------|
| **Random Fourier Features** | 未言及 | カーネル近似の標準手法。Nyström近似の不安定性を解決 |
| **Neural Tangent Kernel** | 未存在 | NNの学習ダイナミクスをカーネルで説明する理論 |
| **Gaussian Process回帰** | GPDM紹介のみ | GPyTorch等で大規模GP可能。ベイズ最適化の標準 |
| **XGBoost/LightGBM** | 未存在 | 特徴選択+非線形性を勾配ブースティングで両立 |
| **Transformer Attention** | 未存在 | softmax(QKᵀ) ≈ カーネル関数。理論的接続確立 |
